## 一、任务概述

在 `前端/学堂/` 目录创建阅读器系统，书籍存储在 NanoHTTPD 的 `assets/books/`，阅读进度与记录页面共用 IndexedDB 数据库。

---

## 二、目录结构

```
前端/学堂/
├── index.html          # 书架页面（书籍列表）
├── reader.html         # 阅读器页面
├── reader.css          # 阅读器样式
├── reader.js           # 阅读器逻辑
├── epub.min.js         # epub.js 核心库
├── epub.min.css        # epub.js 样式
└── books/              # 书籍目录（源文件）
    └── (放置 .epub 文件)

Android/app/src/main/assets/books/
└── (构建时复制 epub 文件)
```

## 四、数据结构

### IndexedDB 表：`reading_progress`

| 字段 | 类型 | 描述 |
|------|------|------|
| id | number | 自动递增 ID |
| book_id | string | 书籍标识（如 `luming-basic.epub`）|
| book_title | string | 书籍标题 |
| progress_cfi | string | epub.js CFI 位置 |
| progress_percent | number | 阅读百分比 (0-100) |
| current_chapter | string | 当前章节标题 |
| last_read | timestamp | 最后阅读时间 |

## 六、技术方案

### 6.1 epub.js 集成

```html
<!-- reader.html -->
<script src="epub.min.js"></script>
<script src="reader.js" type="module"></script>
```

```javascript
// reader.js
const book = ePub('http://localhost:8080/books/luming-basic.epub');
await book.ready;
const rendition = book.renderTo('viewer');
rendition.display(cfi);
```

### 6.2 进度保存时机
- 每翻页后保存
- 关闭阅读器时保存
- 使用 `visibilitychange` 事件监听页面隐藏

### 6.3 目录获取
```javascript
const navigation = await book.loaded.navigation;
const toc = navigation.toc;
```

## 九、风险与注意事项

1. **中文目录支持**：部分 epub 文件可能有中文路径，需要 NanoHTTPD 正确处理 URL 编码
2. **大文件处理**：大尺寸 epub 可能导致 WebView 内存问题，需要分章节加载
3. **CFI 兼容性**：不同版本的 epub.js CFI 格式可能不完全兼容
4. **离线支持**：确保阅读器在完全离线状态下也能正常工作
